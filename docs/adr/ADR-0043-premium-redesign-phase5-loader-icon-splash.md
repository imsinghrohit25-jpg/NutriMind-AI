# ADR-0043: Premium Redesign — Phase 5, Increment 3: Branded Loader, App Icon & Splash

## Status
Accepted (Phase 5, increment 3). Completes Phase 5. Follows ADR-0041 (haptics/governance) and
ADR-0042 (dark/light theming).

## Decision

1. **Branded `AppLoader`** (`core/design_system/components/app_loader.dart`) — a gradient arc
   sweeping in the brand primary (`context.colors.primary`), `AnimationPolicyBuilder`-driven so it
   goes static under OS reduce-motion / when backgrounded, with an optional caption and an
   `AppLoader.centered()` convenience. Replaced the **31 neutral `CircularProgressIndicator`s**
   across the app. The **9 on-button white spinners stay** as plain
   `CircularProgressIndicator(color: Colors.white)` (a solid white ring reads better on a filled
   CTA), and the one **determinate** score gauge in `cart_screen` (a `value:`-driven ring, not a
   loader) is correctly left alone. 4 widget tests.

2. **Production app icon** — generated by `tool/generate_icon.py` (Pillow): a deep-forest diagonal
   gradient squircle (brand `primaryDark→primary→primaryLight`) with a white vesica-piscis leaf and
   saffron midrib/veins clipped inside the leaf. Source `assets/icon/app_icon.png` (1024²) +
   `app_icon_foreground.png` (adaptive foreground). `flutter_launcher_icons` generates Android
   (incl. adaptive `mipmap-anydpi-v26`) and iOS icon sets; config in `pubspec.yaml`. The generator
   is committed so the icon is reproducible, not a mystery binary.

3. **Premium splash** — two layers: (a) `flutter_native_splash` generates the platform pre-engine
   launch screen (brand `#0B120D` background + leaf, light+dark); (b) the in-app `_SplashScreen`
   (shown while the auth stream resolves) is upgraded from a bare spinner to a branded animated
   screen — fixed deep-forest gradient, `NutriMindLogo` (thinking state) scaling+fading in, the
   "NutriMind" wordmark + tagline staggering in via `flutter_animate`, and a white `AppLoader`. The
   8s connectivity-timeout fallback is preserved (now on-brand).

4. **Governance completion** — committing the Jul-13 auth-rebuild files (register/auth_ui/app_intro)
   exposed 12 pre-existing violations to the gate (it only checks tracked files). Fixed: animation
   `Duration`s → `AppMotion` (new `ambient` = 3s tier added for slow float/breathe loops); raw
   `TextStyle(fontSize:)` → `AppType`; brand-hex slide-1 gradient → `AppColors`; a hardcoded toast
   green → `context.colors.success`; the two genuinely decorative (blue/orange) intro-slide gradients
   carry an auditable `// design-governance:ignore:` with reason.

## Consequences
- Every loading state in the app now shows one branded affordance; the app ships a real icon and a
  polished launch experience on both platforms.
- Gate: `flutter analyze` 0 issues; `flutter test` 53/53 (49 prior + 4 loader tests);
  `check-design-governance` 0 violations (85 files). No backend logic touched.
- **Phase 5 (Global Polish) is complete.** The premium redesign (Phases 0–5) is done.
